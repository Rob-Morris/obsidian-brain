# v0.48.5

**Optimise machine scans and test-suite feedback**

**Machine live-runtime scanning**
- `_machine/topology.py` now uses a cheap Python-process gate before enumerating command prefixes during live-runtime detection, so ordinary non-Python processes avoid path allocation and filesystem normalisation work.
- Live-runtime detection now caches runtime parent `realpath()` calls per scan, preserving the existing space-in-path and python-family matching behaviour while avoiding repeated filesystem calls for repeated Python processes.
- Tests cover python-family executable names, paths with spaces, and the cache behaviour in `find_live_brain_runtime_processes()`.

**Test-suite feedback and isolation**
- `Makefile` adds `make test-parallel` as a pytest-xdist fast-feedback path using `-n auto --dist loadscope`; serial `make test` remains the canonical pre-commit gate in `docs/CONTRIBUTING.md`, `docs/contributor/README.md`, and `docs/contributor/agents.md`.
- `tests/conftest.py` centralises `TEMPLATE_VAULT_COPY_IGNORE`, and `tests/test_compile_colours.py` now compiles against an isolated `tmp_path` copy of `template-vault/` so tests do not mutate the checkout.
- `tests/test_mcp_server.py` replaces staleness sleeps with deterministic `_bump_mtime()` updates, while `tests/test_mcp_proxy.py` replaces the proxy give-up test's fixed sleep with a bounded poll loop.
- `tests/test_machine.py` defaults machine-summary tests to a fake no-live-process scan, keeping dedicated live-process tests explicit while avoiding real `ps` calls in unrelated cases.

- Move `BRAIN_INSTALL_REF` to `v0.48.5`; `BRAIN_CLI_VERSION` is unchanged because the CLI dispatch surface did not change.
