# v0.41.1 — Converge managed operational wrappers on the shared runtime handoff

- Route the managed operational direct script surfaces through the shared `_bootstrap/runtime.py` handoff contract: `build_index.py`, `search_index.py`, `construct_benchmark_fixture.py`, `evaluate_search.py`, `compile_router.py`, `compile_colours.py`, `sync_definitions.py`, `shape_printable.py`, `shape_presentation.py`, and `migrate_naming.py` now start in a compatible launcher Python only long enough to enter the canonical managed runtime, then continue substantive work there.
- Keep the wrappers launcher-safe at import time where needed by deferring packageful retrieval imports until after the managed-runtime boundary is settled, so direct script invocation does not fail before the handoff can happen.
- Surface explicit runtime recovery guidance from those wrappers when bootstrap cannot produce a usable managed runtime; the error path now points users at `repair.py runtime` instead of silently continuing in the launcher interpreter.
- Add shared contract coverage in `tests/test_managed_wrapper_handoff.py`, extend existing wrapper CLI tests to model “already inside managed runtime” honestly, and keep the compile-router main-path tests green under the same contract.
- Update the shipped script/runtime documentation to explain the broader managed-wrapper handoff surface and the direct-script baseline for users who do not install the optional `brain` CLI.
