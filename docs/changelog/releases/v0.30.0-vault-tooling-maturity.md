# Vault Tooling Maturity (v0.25.1–v0.30.0)

Shipped: 2026-04-20

## Overview

Brain-core's internal tooling is mature enough that releases, naming, installs, and vault-wide checks work correctly without manual intervention. The naming engine handles lifecycle-dependent filenames, the installer supports multiple clients and auto-discovers vaults, wikilinks are checked inline, and the release artefact model is in place.

## Highlights

- `living/release` artefact type with planned/active/shipped/cancelled lifecycle (v0.28.6)
- Shared naming engine in `_common/_naming.py` with rule selection, render, validate, reverse-parse (v0.28.6)
- Cross-vault registry at `~/.config/brain/vaults` (v0.28.0)
- Inline wikilink checking on `brain_create`/`brain_edit` with `fix_links` convenience flag (v0.28.8)
- Single-file `fix-links` mode via `path` and `links` params (v0.28.8)
- `sync_definitions --status` for read-only type classification (v0.28.7)
- `printables` artefact type with pandoc PDF rendering (v0.28.2)
- Wikilinks standard in `standards/wikilinks.md` (v0.28.8)
- MCP transport packaged under `brain_mcp/` with legacy `mcp/` shims removed (v0.26.0–v0.27.7)
- MCP setup is now optional, non-fatal, and explicitly multi-client for Claude Code and Codex project/user scope (v0.26.2/v0.27.0/v0.27.7/v0.27.8)
- Workspace-aware session bootstrap now surfaces workspace identity and defaults, and the workspace manifest moved to `.brain/local/workspace.yaml` (v0.27.5/v0.27.9)
- `brain_edit` renames artefacts when frontmatter changes flip the active naming rule (v0.28.6)
- `brain_edit` body targeting is explicit: `:entire_body`, `:body_preamble`; legacy `:body` rejected (v0.27.1)
- README, installer, getting-started, and the shipped guide now cover the setup-to-first-artefact path and project-scope activation checks (v0.26.2/v0.27.0/v0.27.8)
- Frontmatter-backed filename rendering shipped in `v0.29.0`. Filenames are now a pure render of frontmatter-selected naming rules, date tokens are driven by declared `date_source`, and runtime reconciliation backfills and relocates artefacts deterministically instead of restamping names from ambient wallclock time.
- Concurrent write safety was hardened across `v0.29.1`–`v0.29.4`. Brain-core now uses unique sibling tempfiles for writes and serialises mutating MCP calls within one server process, reducing the chance of overlapping local mutations corrupting vault state while the broader architectural follow-on is tracked in Brain Shared Mutation Safety.
- Upgrade-time compatibility repair is now explicit. `upgrade.py` gained a versioned `pre_compile_patch` stage in `v0.29.2`, so narrowly scoped fixes can run before the compile gate when an older customised vault would otherwise fail to upgrade.
- Backfilled log workflows now follow the log's subject day rather than the day the file was typed. Since `v0.29.4`, `temporal/logs` filenames and month folders are keyed off `date_source: date`, so retrospective edits stay in the right month.
- Template-vault drift is now enforced through `make sync-template` / `make sync-template-check`, so release hygiene catches stale installed-definition tracking instead of relying on manual inspection.
- Mutation-safety follow-up continued in `v0.29.5`–`v0.29.7`. MCP startup is now strictly non-blocking with phase-logged begin/success/failure markers; the session-mirror write runs through a single long-lived daemon worker with a coalescing `maxsize=1` queue and a 2s `atexit` drain. A `safe_write_via(...)` atomic-write kernel now backs callback-driven serializers — `build_index.py` embeddings and `safe_write_json()` both route through it. Zettelkasten sequencing metadata was renamed `parent + sequence` → `follows + sequence` so lineage stays distinct from ownership semantics.
- Markdown scanning is now uniformly region-aware as of `v0.30.0`. Every brain-core scanner — wikilink extraction, `rename`, `fix-links`, `check`, and the structural landmark helpers (`collect_headings`, `find_section`, `find_body_preamble`) — treats fenced code, inline code, HTML comments, `$$` math, and raw HTML blocks as literal text. A new `replace_wikilinks_in_text` kernel is the single source of truth for region-aware mutation. `extract_wikilinks()` gained a three-mode `literals="exclude"/"include"/"only"` selector defaulting to `"exclude"`. The ad-hoc frontmatter skip in `check_wikilinks_in_file` is removed: YAML property wikilinks (`parent: "foo"`) are now real links matching Obsidian's property-as-link model, so `check` flags broken property-links it previously hid and `rename` / `fix-links` rewrite them on target moves. Slightly breaking for vault-side scripts that relied on the old defaults.
- Migration ledger tracks applied migrations; reinstalling `.brain-core/` no longer replays (v0.26.3)
- Legacy MCP registrations repaired during upgrade (v0.27.6)
- Nested naming prefix when converting temporal artefacts (v0.26.5)
- MCP server shutdown logging tolerates closed stdio pipes (v0.27.2/v0.27.3)
- `sync_definitions --types` restores additive install for uninstalled library types (v0.28.7)
- Upgrade rollback is binary-safe from `v0.29.3`; non-UTF-8 files under `.brain/` or `_Config/` no longer break snapshot or restore during migration.
- Rename and migration paths fail closed on planned target collisions in `v0.29.4`, and post-compile upgrade rollback now snapshots touched artefact roots as well as `.brain/` and `_Config/`.
- Region-aware markdown scanning in `v0.30.0` fixes long-standing inconsistencies: `rename` / `fix-links` / `edit` no longer silently rewrite documentation examples (e.g. `` `old-name` `` inside code blocks); structural scanners no longer misread heading-shaped lines inside HTML comments or `<pre>` blocks as landmarks; and `check` / `extract_wikilinks` / `replace_wikilinks_in_vault` now agree on what counts as a live wikilink.
