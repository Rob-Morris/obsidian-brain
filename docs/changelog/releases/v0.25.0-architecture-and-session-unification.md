# Architecture & Session Unification (v0.24.7–v0.25.0)

Shipped: 2026-04-12

## Overview

This stable milestone covers the work shipped between `v0.24.7` and `v0.25.0`. The headline feature is the unified session bootstrap, but the release line as a whole is defined by deep internal architecture work — DDD-informed decomposition, bounded-context documentation, shared-kernel extraction, and MCP handler modularisation — that makes the codebase structurally ready for the next phase of feature development.

## Highlights

- **`_common` decomposition and shared kernel** (v0.24.7, v0.24.9): the 1,140-line `_common.py` monolith split into 9 focused modules under `_common/` with a tightened facade API; router and artefact helpers promoted into dedicated shared-kernel modules (`_router.py`, `_artefacts.py`), eliminating cross-script helper imports.
- **Bounded contexts and architecture documentation** (v0.24.8): explicit architecture context map covering the 8 bounded contexts with responsibilities and import rules; linked from the architecture overview and mirrored in the script-level module map.
- **Editing improvements** (v0.24.8): explicit whole-section replacement mode (`target=":section:..."`) for `brain_edit`, plus refined targeted-edit policy for wrapper stripping and nested structure.
- **BDD test layer** (v0.24.10): `pytest-bdd` Gherkin scenarios for artefact creation lifecycle, vault compliance checking, and router compilation — thin behavioural coverage exercising existing script APIs.
- **Agent workflow formalisation** (v0.24.11): explicit `trivial` / `small` / `medium` / `large` execution model in the agent contract and shipped bootstrap.
- **MCP handler extraction** (v0.24.12): tool implementation logic extracted from `server.py` into sibling handler modules behind a narrow runtime adapter, aligning the MCP layer with the bounded-context map. MCP documentation refreshed to match.
- **Unified session bootstrap** (v0.25.0): `session.py` now owns one canonical session model rendered as both `brain_session` JSON and a generated markdown mirror at `.brain/local/session.md`. Static core bootstrap content consolidated into `session-core.md`, with `index.md` reduced to a thin bootloader. Core docs surfaced as structured `core_docs` entries with explicit `brain_read` load instructions.
