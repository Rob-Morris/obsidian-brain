# v0.43.1 — Broaden migrate_to_0_43_0.py callout shapes and body conventions

- Broaden `src/brain-core/scripts/migrations/migrate_to_0_43_0.py` to recognise non-canonical legacy closure callouts: `[!warning] Deprecated`, `[!info] Merged`, `[!note] Live planning moved`, and `[!info] Superseded …` shapes with prose (rather than a wikilink) on the heading line. Each pattern carries its own reason keyword so the new callout reads `merged into [[X]]` / `replaced by [[X]]` / `superseded by [[X]]` instead of collapsing to a generic `superseded`.
- Honour body-line conventions (`**Superseded by:** [[X]]`, `**Merged into:** [[X]]`, `**Replaced by:** [[X]]`) and prefer them over the first wikilink found inside a legacy callout — author-written body pointers are more authoritative than callout-block wikilinks, which often lead with parent or related references.
- Extend `tests/test_migrate_to_0_43_0.py` with two new test classes (`TestNonCanonicalCallouts`, `TestBodyConventions`) covering the four added callout shapes and the three body-convention patterns, plus a regression test that a body convention overrides the callout's first wikilink.
- Vaults already migrated to v0.43.0 keep their existing deprecation callouts (migration is idempotent on the presence of `> [!info] Deprecated`); the improvements apply to vaults running the migration for the first time on v0.43.1+.
