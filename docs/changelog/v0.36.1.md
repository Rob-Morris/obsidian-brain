# v0.36.1 — 2026-05-09

**Build vault-driven benchmark tooling and auditable scorecards**

Add the vault-native benchmark tooling that sits on top of the shared semantic retrieval contract. This release turns retrieval validation into a repeatable workflow: build benchmark fixtures from a real vault, audit what the vault can honestly support, and evaluate lexical, semantic, and hybrid retrieval without changing the underlying search implementation.

- Add `construct_benchmark_fixture.py`, which mines lexical / semantic / hybrid / cluster / filter-sensitive cases from a real vault, audits them against live retrieval behaviour, and writes both a benchmark fixture JSON and a machine-readable audit JSON that records admissions, rejections, and bucket shortfall.
- Add `evaluate_search.py`, which runs explicit retrieval modes against a benchmark fixture and reports hit@k, per-mode timing, per-intent summaries, filter-sensitive pass/fail output, basic cluster metrics, and a best-effort expected-winner scorecard.
- Ship the benchmark assets and docs from the start: `.brain-core/scripts/benchmarks/inline-search-benchmark.template.json`, `tests/fixtures/inline_search_benchmark.json`, and the script/reference updates that document local benchmark outputs under `.brain/local/benchmarks/`.
- Fold in the benchmark-slice cleanup that should ship with the feature: the shared `_common._cli` parser helpers keep unknown-flag rejection stable across both CLIs, and `search_index.py` now exposes a shared `dispatch_search(...)` helper so benchmark tooling routes lexical / semantic / hybrid queries through one maintained seam.
