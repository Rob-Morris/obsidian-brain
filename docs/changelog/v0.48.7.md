# v0.48.7

**Verify mutation-time wikilinks against the filesystem index**

**Mutation-time wikilink correctness**
- MCP `brain_create` and `brain_edit` now prepare a filesystem-backed wikilink index before entering the serialized mutation section when `fix_links=true`, then pass that authoritative index to the script layer. This fixes false broken-link warnings and unsafe auto-fix decisions when the retrieval index is stale or missing externally-created artefacts.
- The script-level `fix_links.attach_wikilink_warnings()` contract is explicit again: supplied indexes are used directly, and standalone callers that omit an index still build from the vault on demand.
- This deliberately reverses the v0.35.2 optimisation for the opt-in MCP `fix_links=true` path: correctness for externally-created targets now takes precedence over avoiding the filesystem walk. The walk happens before the mutation lock, so it does not serialize other MCP mutations, but the requesting call can still pay vault filesystem I/O on large or syncing vaults.

**Regression coverage**
- Add MCP coverage for externally-created cargo-barbican-style targets that are present on disk but absent from the in-memory retrieval index.
- Add tests proving the filesystem index is prepared before the mutation context and that the script helper does not perform a hidden fallback walk when given a supplied index.
- Move `BRAIN_INSTALL_REF` to `v0.48.7`; `BRAIN_CLI_VERSION` is unchanged because the CLI dispatch surface did not change.
