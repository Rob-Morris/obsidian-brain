# v0.35.9 — 2026-05-08

**Preview migrations and sync in upgrade --dry-run output**

`upgrade.py --dry-run` reported only the file-copy step (added/modified/removed brain-core files) and silently hid the post-copy work — pre-compile patches, post-compile migrations, and definition sync. A real run could touch dozens of vault files that the dry-run never previewed; one force-upgrade against a clean vault produced 56 file changes after a "Modified: 1 files" preview, including frontmatter rewrites on ~50 logs and several artefact renames. Bug log captured the incident; this fix closes the credibility gap.

- Add `_pending_migrations_summary(...)` helper that mirrors `_run_migrations`' version-filtering logic — same `force` / target / ledger checks — but reads scripts from a caller-supplied directory and returns version metadata only, never executing anything. Accepts a `migrations_dir` override so the dry-run can preview against the source's about-to-be-installed migrations rather than the older set still in the vault.
- Thread `dry_run` through `_post_upgrade_sync` so it calls `sync_definitions(dry_run=True)` instead of executing, and surfaces the result under `sync_preview` (real runs continue to populate `sync_result`). `sync=False` and `artefact_sync: skip` still suppress the preview entirely.
- Extend the `upgrade()` dry-run early-return path to populate `precompile_patch_migrations_preview`, `migrations_preview`, and `sync_preview` keys when applicable. CLI human output now prints "Pre-compile patches that would run", "Post-compile migrations that would run", and a sync preview block — listing migration versions and would-update / would-conflict file roles.
- Update `test_upgrade_dry_run_no_sync` to assert the new "preview without applying" semantics: `sync_result` still absent, vault filesystem still untouched, but `sync_preview` may now appear. Add `test_dry_run_lists_pending_migrations_without_running_them` and `test_dry_run_force_lists_all_migrations_through_new_version` covering both pending and `--force` paths.

Limitation: when an upgrade modifies brain-core library files (artefact taxonomies in `artefact-library/`), the sync preview is computed against the still-old vault library. Same-version `--force` runs (the actual incident scenario) preview accurately because source and vault libraries match. A fully accurate cross-version preview would require staging the source library before invoking `sync_definitions` — left for a follow-up.
