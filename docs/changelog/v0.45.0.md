# v0.45.0

**Resolve the active Brain via a unified ladder with stale-terminates**

- Add `resolve_brain_target()` to `src/brain-core/scripts/_bootstrap/workspace_binding.py`: one pure, launcher-safe resolution ladder — `BRAIN_WORKSPACE_DIR` binding → cwd-walk (`.brain-core/VERSION` vault-self by path, or `.brain/local/workspace.yaml` binding) → `BRAIN_VAULT_ROOT` override → `vault_registry` machine default → error with a bind/set-default cue.
- Classify each binding on its `brain` key: valid resolves and is used; missing falls through; **stale (present but unresolvable) terminates with a prompt and never falls through to a lower rung** — the rule that prevents wrong-Brain routing. A dangling machine default is treated as stale.
- Pin the explicit-anchor case: a `BRAIN_WORKSPACE_DIR` whose binding is missing goes straight to `BRAIN_VAULT_ROOT`, never the cwd-walk, so it cannot cross-resolve a different workspace under the current directory.
- Demote `BRAIN_VAULT_ROOT` below both bindings (previously above the cwd-walk), so a workspace's own binding wins over a legacy override.
- Rewire `src/brain-core/brain_mcp/proxy.py` `_resolve_proxy_target()` onto the shared resolver, keep the resolver pure (no env writes / registration / self-heal), and bump `PROXY_VERSION` to `0.5.0`.
- Record the decision in `docs/architecture/decisions/dd-052-unified-brain-resolution-ladder.md` (extends DD-051); add `tests/test_resolve_brain_target.py` with a table-driven matrix asserting the stale-terminates negatives, the rung-1 anchor pin, vault-root-wins, and resolver purity.
- Move `BRAIN_INSTALL_REF` to `v0.45.0`; `cli/brain` dispatch behaviour and `BRAIN_CLI_VERSION` are unchanged.
