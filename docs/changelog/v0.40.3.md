# v0.40.3 - 2026-05-15

**Converge MCP and repair runtime callers onto canonical `_search`**

- **Move the remaining live runtime retrieval callers onto `_search`.** `src/brain-core/brain_mcp/_server_reading.py`, `src/brain-core/brain_mcp/server.py`, and `src/brain-core/scripts/_repair_runtime.py` now use `src/brain-core/scripts/_search/query.py`, `_search/index.py`, and `_search/assets.py` directly instead of routing retrieval behaviour through `build_index.py` or `search_index.py`.
- **Keep the top-level search scripts as public wrappers only.** `src/brain-core/scripts/build_index.py` and `src/brain-core/scripts/search_index.py` remain supported script entry surfaces, while `docs/architecture/overview.md`, `docs/functional/scripts.md`, and `src/brain-core/scripts/README.md` now describe the staged wrapper boundary honestly.
- **Lock the wrapper boundary down with structural tests.** `tests/test_mcp_server.py` now patches canonical `_search` owners, and `tests/test_search_modules.py` walks `src/brain-core/brain_mcp/` plus `src/brain-core/scripts/` to assert that no live runtime module imports `build_index` or `search_index` for retrieval behaviour.
