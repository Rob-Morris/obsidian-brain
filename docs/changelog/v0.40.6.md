**Make unreadable retrieval sources fail explicitly and consistently**

- `src/brain-core/scripts/_search/errors.py` adds `UnreadableRetrievalSourceError`, `CompiledRouterUnavailableError`, and `RetrievalPersistenceError` so `build_index.py`, `_search.resource.search_resource(...)`, and `construct_benchmark_fixture.py` fail with typed retrieval-state errors instead of silent skips or stale-state ambiguity.
- `src/brain-core/scripts/_search/index.py`, `src/brain-core/scripts/_search/assets.py`, and `src/brain-core/scripts/build_index.py` now treat unreadable source bodies, compiled-router embeddings drift, and retrieval-index persistence failures as explicit boundary failures; `tests/test_build_index.py` covers the direct wrappers and the CLI output contract.
- `src/brain-core/brain_mcp/server.py`, `src/brain-core/brain_mcp/_server_readiness.py`, and `src/brain-core/brain_mcp/_server_reading.py` thread those typed failures through shared `index_error` state so index-backed `brain_search`, `brain_list`, and `brain_process` return the real rebuild error until a successful refresh clears it; `tests/test_mcp_server.py` covers startup, rebuild, and stale-index preservation behavior.
- `src/brain-core/scripts/_search/snippet.py` keeps snippet generation as a presentation-only soft degrade by omitting `snippet` for unreadable rereads, while `src/brain-core/scripts/construct_benchmark_fixture.py` now fails explicitly on unreadable benchmark sources; `tests/test_search_index.py` and `tests/test_construct_benchmark_fixture.py` pin both behaviors.
