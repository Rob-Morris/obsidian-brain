# v0.44.2

**Summary:** Bump PROXY_VERSION and BRAIN_CLI_VERSION for workspace routing

- `src/brain-core/brain_mcp/proxy.py` now bumps `PROXY_VERSION` to `0.4.1`, so a vault upgraded to the new workspace-binding and typed-local authoritative-registry line does not report the shipped proxy as `0.4.0+modified`.
- `cli/brain` now bumps `BRAIN_CLI_VERSION` to `1.0.1` for the CLI-side workspace-routing/bootstrap hardening and keeps `BRAIN_INSTALL_REF` pinned to `v0.44.2` so `brain install` still downloads the matching `install.sh` release.
- `.canaries/pre-commit.md`, `docs/CONTRIBUTING.md`, `docs/contributor/README.md`, `docs/contributor/agents.md`, and `docs/functional/cli.md` now teach the explicit version-surface checks for `PROXY_VERSION`, `BRAIN_CLI_VERSION`, and `BRAIN_INSTALL_REF` so future pre-commit passes catch this class of drift before propagation.
- `README.md`, `docs/user/getting-started.md`, `docs/user/user-reference.md`, `docs/functional/scripts.md`, `docs/architecture/overview.md`, and `src/brain-core/scripts/README.md` now describe the shipped `setup workspace` / `configure mcp` surface, the launcher-safe `setup.py` baseline, and the `vault_registry.py --resolve BRAIN_ID` contract consistently.
