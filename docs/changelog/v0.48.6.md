# v0.48.6

**Add native Windows install and MCP smoke support**

**Native Windows runtime support**
- `_common/_file_lock.py` now owns cross-platform lock acquisition for `vault_registry.py` and `_machine/discovery.py`, using `fcntl.flock` on POSIX and `msvcrt.locking` on Windows without importing POSIX-only modules at startup.
- `_common/_venv.py` now resolves Windows managed-runtime interpreters as `Scripts/python.exe`, normalises win32 executable identity with `ntpath.normcase()`, and keeps POSIX launch-path identity on `abspath()` so venv symlink boundaries are preserved.
- `_bootstrap/runtime.py` uses a win32 `subprocess.run()` handoff path instead of `os.execve()`, so MCP startup keeps the client stdio channel attached on native Windows.
- `_common/_paths.py`, `_common/_filesystem.py`, and `obsidian_cli.py` now use platform-aware config and temp paths, including `%APPDATA%\brain` for win32 machine-local config.
- `_common/_shell.py`, `_repair_common.py`, `session.py`, `doctor.py`, `configure.py`, `upgrade.py`, `_bootstrap/mcp_transport.py`, and `_machine/maintenance.py` now route recovery and guidance commands through centralised platform-aware quoting.
- Claude SessionStart hooks on Windows now use an explicit PowerShell shell form with PowerShell-safe argv quoting, so vault paths containing shell metacharacters are handled correctly.

**Installer core and Windows launcher**
- `install.py` is the shared Python installer core for fresh and existing-vault installs: it scaffolds the vault, installs `.brain-core/`, provisions the managed runtime, registers MCP through `_bootstrap/mcp_transport.py`, and emits lifecycle results with partial-failure recovery notes.
- `install.sh` now acts as a POSIX launcher that still owns source acquisition, upgrade delegation, uninstall, and prompts, but hands non-upgrade install policy to `install.py`.
- `install.ps1` and `install.cmd` add the native Windows user launcher path, including Python 3.12+ discovery, MCP scope prompting, existing-default warning text, and the same `install.py` handoff as POSIX.
- `_bootstrap/mcp_transport.py` now owns shared MCP follow-up notes for installer and configure flows, so Claude/Codex verification guidance is not duplicated across shell launchers.

**Regression coverage**
- `tests/test_common_file_lock.py`, `tests/test_venv.py`, `tests/test_managed_wrapper_handoff.py`, `tests/test_common_shell.py`, `tests/test_install_core.py`, `tests/test_install_script.py`, `tests/test_install_script_vault_registry.py`, and `tests/test_configure.py` cover the cross-platform lock, venv, handoff, quoting, installer-core, shell-launcher, and MCP-guidance contracts.
- `.github/workflows/windows-smoke.yml` adds a minimal `windows-latest` smoke path that runs `tests/test_windows_user_smoke.py` only, proving import, `install.ps1`, installed MCP startup, and one `brain_read(resource="environment")` round trip without making the full suite a Windows contributor gate.
- Move `BRAIN_INSTALL_REF` to `v0.48.6`; `BRAIN_CLI_VERSION` is unchanged because the CLI dispatch surface did not change.
