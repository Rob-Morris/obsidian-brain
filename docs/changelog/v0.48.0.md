# v0.48.0

**Harden resolution sign-off: single vault predicate and anchor hard-error**

- Single-source the narrow vault predicate: move `is_brain_vault` (keys on `.brain-core/VERSION`) into `src/brain-core/scripts/_common/_vault.py` and import it in both `_bootstrap/workspace_binding.py` and `_bootstrap/mcp_state.py`, removing the byte-identical `mcp_state.is_vault_root` twin. The broad `_common.is_vault_root` (also matching `AGENTS.md`) is kept, distinctly named, for CLI discovery; `upgrade.py` keeps its own bootstrap-time copy by design.
- Hard-error an explicit anchor with a missing binding (design Decision #2): when `BRAIN_WORKSPACE_DIR` is set, its binding is absent, and no `BRAIN_VAULT_ROOT` resolves, `resolve_brain_target` raises instead of falling through to the machine default — which could serve a different Brain than the workspace was bound to.
- Make the stale-binding message actionable: distinguish "Brain id not in the registry" from "registered but its vault is missing/moved" (`_stale_binding_detail`), reusing the brain id now returned by `_classify_workspace_binding` (no second manifest read, no `assert` crutch).
- Narrow the `heal_legacy_config` `except` clauses to the raisable set so a logic bug surfaces to the `resolve_and_heal` backstop rather than being masked as a routine heal failure; the outer best-effort backstop is retained so a heal bug cannot crash proxy startup.
- Surface a legacy `BRAIN_VAULT_ROOT` as an informational `brain doctor` finding (design Decision #6) via `_bootstrap/diagnostics.py` — no proactive notify, no effect on exit code.
- `install.sh`: surface an explicit `--id` collision instead of swallowing it (`registry_update`), and simplify `--id` parsing to a single pass.
- Remove the dead `find_bound_workspace_dir`; bump `PROXY_VERSION` to `0.5.3` so the resolution-precedence change takes effect on the next proxy restart; move `BRAIN_INSTALL_REF` to `v0.48.0`.
- Record the corrections as sign-off refinements in `docs/architecture/decisions/dd-053-explicit-vault-self-flag-and-unified-predicate.md` (now also extends DD-049): the single-source predicate, the Decision-#2 hard-error, and that DD-049's dispatch surface includes `setup`.
- Tests: the anchor hard-error, the rung-2 walk-stop (no climb to a parent workspace), the rung-4 default writing no `workspace.yaml`, the stale-message distinction, the `proxy.main` env-capture-before-mutation guard, the doctor finding, and the `--id` collision surfacing (`tests/test_resolve_brain_target.py`, `tests/test_workspace_binding_heal.py`, `tests/test_mcp_proxy.py`, `tests/test_doctor_machine.py`, `tests/test_install_script_vault_registry.py`).
