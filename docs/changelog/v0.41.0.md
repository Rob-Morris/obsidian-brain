# v0.41.0 - 2026-05-19

**Extract shared `_bootstrap` runtime seams for lifecycle entrypoints**

- New package `src/brain-core/scripts/_bootstrap/` now owns the shared runtime handoff and launcher-safe diagnostic seams in `runtime.py` and `diagnostics.py`, including the canonical `BRAIN_MANAGED_RUNTIME` / `BRAIN_BOOTSTRAP_SUMMARY` env-var contract.
- `src/brain-core/scripts/init.py`, `repair.py`, `configure.py`, `session.py`, and `check.py` now converge on that shared bootstrap/runtime owner instead of carrying wrapper-local launcher discovery and managed-runtime signalling.
- `src/brain-core/scripts/check.py` and `src/brain-core/scripts/_repair_runtime.py` now split launcher-safe runtime/MCP/registry diagnostics from the managed semantic diagnostic tail, so `check.py` can route through a small bootstrap seam without relying on incidental lazy semantic imports.
- `src/brain-core/scripts/session.py` now preserves SessionStart survivability explicitly: normal execution hands off into the managed runtime, while managed-runtime bootstrap failure emits a minimal degraded JSON payload with shaped recovery guidance instead of failing silently.
- `src/brain-core/scripts/vault_registry.py`, `workspace_registry.py`, and the updated tests under `tests/test_init.py`, `test_session.py`, `test_check.py`, `test_configure.py`, and `test_repair.py` now pin the bootstrap-plane contract more honestly and verify the new wrapper behaviour directly.
