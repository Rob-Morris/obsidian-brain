# v0.40.5 - 2026-05-16

**Split _search retrieval into focused modules and remove lazy seams**

- **Split the canonical retrieval owners inside `src/brain-core/scripts/_search/`.** `src/brain-core/scripts/_search/query.py` is deleted, and its former mixed responsibilities now live in `lexical_query.py`, `semantic_query.py`, `hybrid_query.py`, `mode.py`, `resource.py`, and `snippet.py`, with `search_index.py`, `evaluate_search.py`, `construct_benchmark_fixture.py`, `brain_mcp/_server_reading.py`, and `brain_mcp/server.py` calling those focused owners directly.
- **Make `SearchFilters` the single retrieval filter contract.** `src/brain-core/scripts/_search/filters.py` now drives artefact filtering across lexical, semantic, and hybrid retrieval, while `src/brain-core/scripts/evaluate_search.py`, `src/brain-core/scripts/construct_benchmark_fixture.py`, and `src/brain-core/brain_mcp/_server_reading.py` build and pass `SearchFilters` explicitly instead of re-threading loose `type` / `tag` / `status` trios.
- **Remove the lazy semantic proxy seam and harden semantic availability checks.** `src/brain-core/scripts/_search/_lazy.py` is deleted; `_search/semantic_query.py` now exposes `encode_query_or_unavailable(...)` and raises the canonical missing-sidecar error directly, and `src/brain-core/scripts/_semantic/runtime.py` now owns `embeddings_meta_matches_current_router(...)` so router-hash validation no longer reaches behind the semantic runtime boundary.
- **Restore a clean data flow for embeddings refresh without reviving the old doc smuggle.** `src/brain-core/scripts/_search/index.py` now populates an optional `embedding_parts_by_path` cache using `src/brain-core/scripts/_search/document_parts.py`, `src/brain-core/scripts/_search/assets.py` consumes that cache during `build_embeddings(...)` / `persist_retrieval_outputs(...)`, and `src/brain-core/brain_mcp/server.py` carries the same cache through live rebuild paths so semantic refresh no longer re-opens every artefact body after index construction.
- **Remove dead MCP drift and tighten the verification surface.** `src/brain-core/brain_mcp/server.py` drops the orphaned `_transform_cli_results` / `_fmt_search` / `_fmt_list` copy, `src/brain-core/scripts/_taxonomy_descriptions.py` pulls taxonomy reads out of `_search.index`, and the new contracts are locked in by `tests/test_search_modules.py`, `tests/test_build_index.py`, `tests/test_search_index.py`, `tests/test_evaluate_search.py`, `tests/test_construct_benchmark_fixture.py`, and `tests/test_mcp_server.py`.
