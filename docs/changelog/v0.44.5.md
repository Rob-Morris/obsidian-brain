# v0.44.5

**Add default Brain pointer and explicit-id support to vault_registry**

- Add a machine default Brain pointer to `src/brain-core/scripts/vault_registry.py`, stored in a separate `$XDG_CONFIG_HOME/brain/default` file (a single Brain ID) so the `vaults` row format is untouched: `get_default()` (unlocked, pure), `set_default()` (validates the id is a known local entry), and `clear_default()`.
- Extend `register()` with an optional explicit `brain_id`: a free id on an unregistered path creates the entry, an id already mapping to this path is a no-op, and an id bound to a different path — or a path already registered under another id — raises `RegistryConflictError`. Re-keying an existing vault is intentionally unsupported (it would orphan workspaces bound to the old id); that stale-cure policy is deferred to the migration layer.
- Clear a dangling default pointer atomically within the same lock when `unregister()` or `prune()` removes the Brain it points at.
- Surface the new capability on the CLI: `--register --id ID`, `--set-default`, `--get-default`, `--clear-default`, and a `default` flag in `list_entries()` / `--list`.
- Add coverage in `tests/test_vault_registry.py` for the default lifecycle, the four explicit-id branches, slug validation, `backfill()` idempotency, `prune`/`unregister` default-clearing, and the new CLI flags.
- Move `BRAIN_INSTALL_REF` to `v0.44.5`; `cli/brain` dispatch behaviour and `BRAIN_CLI_VERSION` are unchanged.
