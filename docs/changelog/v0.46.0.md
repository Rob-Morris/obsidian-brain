# v0.46.0

**Self-heal legacy Brain configs on proxy startup, bind before register**

- Add `heal_legacy_config()` and `resolve_and_heal()` to `src/brain-core/scripts/_bootstrap/workspace_binding.py`: a best-effort, idempotent, missing-only self-heal that runs in the MCP proxy startup after a successful (non-stale) resolution. Independent triggers — self-register the vault when resolved by being inside it; complete a missing project-reg binding from `BRAIN_VAULT_ROOT`; and seed the machine default from a legacy user-reg `BRAIN_VAULT_ROOT` (validated, only when no default is set).
- Seed the user-reg default from the captured `BRAIN_VAULT_ROOT` env value, never the resolved target, so operating inside a different Brain cannot set the wrong machine default. `brain_mcp/proxy.py` `main()` captures the env signals once, before its own `os.environ` mutation, and passes them to `resolve_and_heal`.
- Guarantee the heal never runs on a stale binding: `resolve_brain_target` raises first, so a broken registry surfaces rather than being silently healed — verified by a byte-identical registry/manifest snapshot after a stale raise.
- Flip the write-ordering in `_bootstrap/mcp_transport.py` `apply_mcp_transport_action`: converge the workspace binding *before* writing any MCP registration, so an interrupted setup never leaves a `BRAIN_WORKSPACE_DIR` registration without a binding. The single shared engine covers `init.py` and `install.sh`.
- Remove the dead `_resolve_proxy_target()` from `brain_mcp/proxy.py` (zero callers; it re-read `os.environ` post-mutation, a latent wrong-default footgun); bump `PROXY_VERSION` to `0.5.1` so the heal activates on the next proxy restart.
- Confirm `build_mcp_config` writes no `BRAIN_VAULT_ROOT` (unchanged) and add `tests/test_workspace_binding_heal.py` covering every heal branch, co-fire, seed-source, the dangling guard, the stale-no-heal hazard, and the write-ordering invariant.
- Move `BRAIN_INSTALL_REF` to `v0.46.0`; `cli/brain` dispatch behaviour and `BRAIN_CLI_VERSION` are unchanged.
