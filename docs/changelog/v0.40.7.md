# v0.40.7 - 2026-05-17

**Move semantic asset refresh into `_semantic` and `_lifecycle`**

- `src/brain-core/scripts/_semantic/assets.py` now owns semantic sidecar build, persistence, and clear mechanics, while `src/brain-core/scripts/_lifecycle/retrieval_assets.py` owns the combined router + lexical + semantic refresh flow that used to sit under `src/brain-core/scripts/_search/assets.py`.
- `src/brain-core/scripts/_lifecycle/document_parts.py` and `src/brain-core/scripts/_lifecycle/retrieval_errors.py` now hold the shared retrieval-side value and error types above `_search` and `_semantic`, and `src/brain-core/scripts/_search/assets.py`, `_search/document_parts.py`, and `_search/errors.py` are removed.
- `src/brain-core/scripts/build_index.py`, `src/brain-core/scripts/_semantic/provision.py`, and `src/brain-core/brain_mcp/server.py` now consume the new boundary directly, including the MCP loaded-state embeddings refresh path, so strict semantic refresh, generic sidecar clearing, and typed index-error behavior all route through `_lifecycle.retrieval_assets` instead of reaching into `_semantic` ad hoc.
- `src/brain-core/scripts/_semantic/model.py` now treats missing `sentence_transformers` as a typed `SemanticRuntimeUnavailableError` instead of a snapshot-load failure, and `provision_semantic_model(...)` no longer retries that deterministic runtime break by redownloading the model snapshot.
- `docs/architecture/overview.md`, `docs/functional/scripts.md`, `docs/user/user-reference.md`, and `src/brain-core/scripts/README.md` now describe the shipped ownership boundary honestly; `tests/test_search_modules.py` adds an AST guard that forbids re-importing the retired `_search` refresh modules.
