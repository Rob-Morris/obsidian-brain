# v0.36.2 — 2026-05-09

**Tune hybrid retrieval around lexical anchors, semantic rescue, and Brain title champions**

Tighten hybrid retrieval so it behaves more like the practical default the semantic stack has been aiming for, especially on exact-anchor queries, title-shaped prompts, and nearby conceptual neighbours. This release keeps the existing RRF window and deferred bonus-pruning decisions intact while making the shipped hybrid heuristics explicit and better covered end to end.

- Raise `RRF_SEMANTIC_WEIGHT` to `1.0` without changing the candidate window or reciprocal-rank defaults, so the semantic leg contributes equally before the targeted hybrid bonuses apply.
- Use `LEXICAL_ANCHOR_RE` to short-circuit obvious exact-anchor queries in `search_hybrid(...)`, keeping version strings and ticket-like identifiers on the lexical path instead of diluting them through fusion.
- Add the semantic-champion, lexical-title-champion, and semantic-rescue score nudges inside `_fuse_rrf(...)`, including the explicit Brain-only title-prefix rule that strips only the shipped `Brain` product namespace from first-party titles like `Brain MCP Server`.
- Add both helper-math coverage and end-to-end `search_hybrid(...)` tests for the champion/rescue paths so the real lexical/semantic wiring is exercised without turning this slice into a bonus-pruning pass.
