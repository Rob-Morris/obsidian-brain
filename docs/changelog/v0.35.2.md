# v0.35.2 — 2026-05-04

**Keep brain_edit and brain_create fix_links off vault-wide walks**

`brain_edit` and `brain_create` with `fix_links=true` previously derived wikilink state by walking the whole vault during the post-mutation scan. On iCloud-backed vaults that turned a single-file mutation into unbounded filesystem I/O and could stall the MCP path under sync contention. This release switches the write path to state-derived file indexes, then hardens the mutation-only index-readiness and cache behaviour so the bounded-cost path stays bounded across rescans.

- Add `file_index_from_documents()` in `src/brain-core/scripts/_common/_wikilinks.py` and `file_index_from_state()` in `src/brain-core/brain_mcp/_server_artefacts.py`, then thread `file_index` through `src/brain-core/scripts/create.py`, `src/brain-core/scripts/edit.py`, and `src/brain-core/scripts/fix_links.py` so MCP mutations stop calling the legacy full-vault walker.
- Add `_ensure_mutation_index_ready()` and shared `_try_rebuild_index()` handling in `src/brain-core/brain_mcp/server.py`, then route `brain_create` and `brain_edit` through `require_fix_links_file_index()` in `src/brain-core/brain_mcp/_server_artefacts.py` and `src/brain-core/brain_mcp/_server_runtime.py` so write-time `fix_links` drains `_index_pending` and handles `_index_dirty` without the TTL-driven read/search freshness sweep.
- Move request-local file-index overlay helpers into `src/brain-core/scripts/_common/_wikilinks.py`, add `build_vault_basename_index()`, and make `ensure_complete_file_index_basenames()` persist the lazy asset-basename cache so post-fix rescans do not repeat the same basename walk.
- Simplify structural-boundary metadata in `src/brain-core/scripts/_common/_markdown.py` to `next_boundary_raw`, and extend `tests/test_edit.py` so section replacement also rejects shallower trailing boundary duplication.
- Add and extend regression coverage in `tests/test_common_wikilinks.py`, `tests/test_fix_links.py`, `tests/test_create.py`, `tests/test_edit.py`, `tests/test_markdown_structural.py`, `tests/test_mcp_proxy.py`, and `tests/test_mcp_server.py` to cover the mutation-only readiness path, cached rescans, shallower boundary rejection, and preserved unread NDJSON lines.
