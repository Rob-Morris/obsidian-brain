# v0.39.0 — 2026-05-11

**Converge central-runtime resolution across every entry point**

This slice finishes the central-runtime story started in `v0.37.0` and hardened in `v0.38.0`. Every supported managed-runtime entry point now delegates to one owner in `_common/_venv.py`, and the follow-up fixes that made that convergence safe for end users are folded into the same release.

- **Single runtime owner across entry points** — `_common._venv.resolve_or_provision_central_venv` now owns resolve, reuse, sync, and create semantics for the `brain` CLI dispatch path, `_lifecycle_common.bootstrap_managed_runtime`, `repair.py`, `configure.py`, and `upgrade.py`. Exact-tag lookup still wins first; otherwise any compatible same-hash runtime is reused before a new exact-tag runtime is created. `tests/test_central_runtime_convergence.py`, `tests/test_configure.py`, and `tests/test_repair.py` lock that behaviour in across the Brew-churn fixture.
- **Repair reused runtimes in place during upgrade** — when `upgrade.py` requests dependency sync and the reused runtime lacks or stales `.brain-deps-installed`, `_common/_venv.py` now repairs that existing runtime in place instead of returning `RUNTIME_REUSED` too early. `tests/test_central_runtime_convergence.py` and `tests/test_upgrade.py` cover the forced-sync reused-runtime path.
- **Keep CLI installs explicit** — `src/brain-core/scripts/upgrade.py` now refreshes only existing `brain` installs instead of creating a new machine-level binary during vault upgrade, and `cli/brain` pins `brain install` to the shipped `v0.39.0` installer ref instead of floating GitHub `main`. `tests/test_upgrade.py` and `tests/test_brain_cli.py` cover both ownership boundaries.
- **Contributor semantic setup is repo-local only** — `Makefile install-semantic`, `docs/CONTRIBUTING.md`, `tests/test_search_index.py`, and `tests/test_semantic_model.py` now treat semantic extras as repo-local contributor dependencies and skip NumPy-only assertions when the optional stack is absent, keeping that setup separate from installed-vault lifecycle behaviour.

`DD-048` now carries both the `v0.38.0` lookup clarification and the `v0.39.0` entry-point convergence note. The remaining deferred item is launcher-resolver cleanup: `install.sh`, `cli/brain`, `upgrade.py`, and `_lifecycle_common.find_repair_launcher` still choose launchers independently, but they now feed one orchestrator so the user-visible duplicate-runtime risk is closed.
