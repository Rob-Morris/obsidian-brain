# v0.35.6 — 2026-05-06

**Codify type extension procedure to match brain-core behaviour**

The `adding-living-type.md` and `adding-temporal-type.md` standards under `src/brain-core/standards/extending/` were six-bullet checklists that elided most of what every shipped taxonomy already does — required sections, parser-required formats, the type-key derivation rule, hub-pattern integration, the local-vs-upstream contract, and the artefact-library bundle layout. Audited `compile_router.py`, `sync_definitions.py`, `check.py`, and existing taxonomies; folded the implicit conventions into both standards so an agent or contributor following them produces a type the tooling can compile, validate, and sync.

- Expand `src/brain-core/standards/extending/adding-living-type.md` and `adding-temporal-type.md` to document: type-key derivation from folder name; the *type identifier* (`living/{plural}`) vs *frontmatter type* (`living/{singular}`) split tracked by the compiler; required vs optional taxonomy sections with the exact heading text the parser matches; `## Naming` simple/advanced forms; `## Frontmatter` YAML-code-block format; `## Template` wikilink contract; `## Trigger` first-line=condition / rest=detail with the router.md condition merged at compile time; `## On Status Change` transition syntax; the `configured` flag for taxonomy-less folders; the validation step (`check.py --actionable`); the local-vs-upstream contract via `sync_definitions.py` three-way hash comparison and `.brain/tracking.json`; the artefact-library bundle layout (`manifest.yaml` / `schema.yaml` / `taxonomy.md` / `template.md` / `README.md`) including the hand-rolled YAML manifest schema and the schema.yaml constraint vocabulary; the clarification that `router_trigger` in the manifest is informational rather than auto-installed; cross-references to the keys, hub-pattern, naming-conventions, subfolders, archiving, linking, provenance, and wikilinks standards.
- Align the matching procedure in `docs/user/system-guide.md` with the standards: drop the redundant "Reference standards" step (now folded into taxonomy authoring guidance) and add a Validate step using `check.py`, keeping both lists at seven steps and pointing readers at the standards for the full procedure.
- Align the one-liner extension procedures in `docs/contributor/specification.md` with the standards: drop the "reference standards (provenance/archiving)" sub-step (now described as a property of the taxonomy rather than a sequential step) and add the `check.py` validate step.
