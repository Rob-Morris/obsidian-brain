# v0.42.1 — Add portable lexical retrieval wrappers alongside the managed search surfaces

- Add `src/brain-core/scripts/build_lexical_index.py` as the portable lexical-only index builder over `_search.index`, so no-network or no-runtime callers can rebuild `.brain/local/retrieval-index.json` without pulling in semantic/runtime handoff behaviour.
- Add `src/brain-core/scripts/search_lexical.py` as the portable lexical-only query surface over `_search.lexical_query`, sharing the same lexical index format and filter semantics without exposing semantic or hybrid mode flags.
- Document the retrieval split across `docs/functional/scripts.md`, `docs/user/user-reference.md`, `src/brain-core/scripts/README.md`, and `src/brain-core/guide.md` so the portable lexical wrappers and the managed full-search wrappers have distinct, explicit contracts.
- Verify the new portable wrappers directly in `tests/test_build_lexical_index.py` and `tests/test_search_lexical.py`, alongside the existing managed wrapper tests for `build_index.py` and `search_index.py`.
