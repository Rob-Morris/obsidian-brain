- Add `src/brain-core/scripts/setup.py` as the public `brain setup workspace` owner and extend `src/brain-core/scripts/configure.py` with `workspace binding`, `workspace metadata`, `workspace bootstrap`, and `mcp` surfaces so workspace binding and transport policy are no longer one command.
- Introduce `src/brain-core/scripts/_bootstrap/workspace_binding.py` as the single launcher-safe owner for `.brain/local/workspace.yaml` convergence, legacy-path migration, symbolic Brain-ID resolution, and shared binding validation across `session.py`, `init.py`, and `_bootstrap/mcp_state.py`.

- Add `docs/architecture/decisions/dd-051-workspace-owned-binding-and-workspace-aware-mcp.md` and restore `dd-023`, `dd-039`, and `dd-040` to immutable-body historical records with forward links only, so the DD layer matches policy again.
- Cover the cutover in `tests/test_setup.py`, `tests/test_configure.py`, `tests/test_init.py`, `tests/test_session.py`, and `tests/test_brain_cli.py`, including guided rebind, structured `configure mcp`, and workspace-aware target resolution.

Migration:
- Use `brain setup workspace` for first-time workspace binding instead of teaching `init.py` as the public noun.
- Use `brain configure mcp` for explicit transport policy and `brain configure workspace ...` for targeted binding, metadata, and bootstrap changes.
- Treat legacy Brain-bound MCP config as transitional and reconfigure onto the workspace-aware route when you next touch an existing install.
