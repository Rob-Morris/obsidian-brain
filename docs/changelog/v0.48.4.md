# v0.48.4

**Generate distinctive artefact keys before random suffixes**

- `create.py` now generates living artefact keys with `_common.derive_distinctive_slug()` so new artefacts use the clearest free title-derived key before falling back to a random suffix. For example, `Pistols at Dawn` now gets `pistols-dawn` when that key is free.
- `edit.py` now uses the same distinctive-key selection when converting an artefact into a living type, keeping create and convert behaviour aligned.
- `_common/_slugs.py` drops the unused `generate_contextual_slug()` helper and `_common` re-export now that all runtime key generation routes through `derive_distinctive_slug()`.
- `_common/_slugs.py` now guarantees random suffixes always include a letter, so alpha-free titles such as `1984` produce valid generated keys by construction instead of depending on post-hoc rejection.
- `standards/keys.md` now documents the collision policy: generated keys try the distinctive two-word form, then the strongest single keyword, and only use the random 3-character suffix after those candidates are taken.
- Tests cover unsuffixed generated keys, single-keyword collision fallback, random suffix fallback, the suffix letter guarantee, numeric-title validity, sentinel fallback, and retry exhaustion across helper, create, and convert flows (`tests/test_common_slugs.py`, `tests/test_create.py`, `tests/test_edit.py`).
- Move `BRAIN_INSTALL_REF` to `v0.48.4`; `BRAIN_CLI_VERSION` is unchanged because the CLI dispatch surface did not change.
