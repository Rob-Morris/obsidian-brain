# v0.35.4 — 2026-05-04

**Restore start_shaping.py CLI bootstrap via load_compiled_router**

The `start_shaping.py` CLI path was calling `compile_router.load(...)`, which does not exist, so direct script use crashed before shaping could create or append its transcript. This release realigns the script with the existing compiled-router cache seam and adds regression coverage for the public bootstrap path.

- Update `src/brain-core/scripts/start_shaping.py` so its CLI entrypoint loads `.brain/local/compiled-router.json` through `load_compiled_router(...)` and exits with a CLI-friendly error if the compiled router is missing.
- Add regression coverage in `tests/test_start_shaping.py` for first-run CLI transcript creation/linking and same-day CLI transcript append behaviour.
- Verify the repaired script path with `tests/test_start_shaping.py` and the shared MCP action surface with `tests/test_mcp_server.py -k start_shaping`.
