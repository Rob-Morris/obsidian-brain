# v0.35.1 — 2026-05-04

**Reject duplicate-boundary `brain_edit` section replacement bodies**

`brain_edit` with `scope='section'` previously accepted bodies whose final heading line restated the original next-sibling boundary heading, silently producing a file with the boundary heading written twice in a row. This release rejects that pattern with an actionable error, then hardens the resolver/validator contract so the boundary guard and related heading-body validation share one stable target shape.

- Extend `resolve_structural_target` to expose `level`, `next_sibling_raw`, and `next_sibling_level` unconditionally, so heading validators use the resolver's own section metadata instead of re-parsing or conditional key checks.
- Update `_validate_heading_section_replacement` to reject when the body's last heading line equals the boundary at the target's level. Bodies ending in prose, deeper subsections, or interior same-level headings with trailing content are still accepted.
- Update `_validate_heading_body_payload` and related validator code to read `resolved["level"]` directly, drop defensive `.get()` patterns on invariant keys, and document the upstream heading-collection invariant explicitly.
- Strengthen `tests/test_edit.py` and `tests/test_markdown_structural.py` with ordering assertions plus edge cases covering no internal headings, level-mismatch acceptance, mid-body plus trailing duplicate rejection, interior-heading boundary matches, and the no-next-sibling case.
