# v0.49.1

**Model MCP config freshness as an explicit state machine**

- Add `brain_mcp/_server_config_state.py` with `ConfigFresh`, `ConfigProbeError`, and `ConfigLoadError` so transient probe failures and sticky load failures cannot drift across parallel globals.
- Route MCP config refresh through one probe → decide → load → commit primitive in `brain_mcp/server.py`; signatures are computed once per refresh, load runs under `_config_lock`, and successful reloads commit state before publishing side effects.
- Add `config.config_input_paths()` and `config.load_config_from_paths()` so the MCP server can detect config input changes without rediscovering config paths during load.
- Add `config.ConfigError` for parseable but structurally invalid config, so malformed `vault` / `defaults` shapes fail closed as config errors instead of crashing startup or surfacing as unexpected tool errors.
- Refresh config before `brain_session` authentication and profile enforcement; `brain_init(debug=true)` now reports `debug.config_error`, and removed active profiles fail closed until the operator reruns `brain_session` or fixes config.
- Keep background derived work on last-good config during transient probe errors via `_derived_work_config()`, while guarded MCP tools continue to use the fail-closed `_current_config()` accessor.
- Update `docs/functional/config.md` and `docs/functional/mcp-tools.md` for live config reload, config-error fail-closed behaviour, and active-profile removal semantics.
- Move `cli/brain` `BRAIN_INSTALL_REF` to `v0.49.1`; `BRAIN_CLI_VERSION` is unchanged because the CLI dispatch surface did not change.
