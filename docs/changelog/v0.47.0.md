# v0.47.0

**Add vault-self MCP mode and exclusive install scope choice**

- Add a rung-1 explicit vault-self flag to `resolve_brain_target` (`src/brain-core/scripts/_bootstrap/workspace_binding.py`): when `BRAIN_WORKSPACE_DIR` points at a vault root, resolve vault-self by path without a binding lookup, so a this-vault-only registration resolves deterministically regardless of the proxy's cwd.
- Unify the vault-root predicate: add the public `is_brain_vault` (keys on `.brain-core/VERSION`) and route every resolution, heal, and refuse-guard decision — plus `init.py --skip-mcp`'s skip-binding-for-a-vault check — through it, replacing the broad `_common.is_vault_root` (which also matches `AGENTS.md`-only directories and is now kept only for CLI discovery). Fixes an `AGENTS.md`-bearing workspace (e.g. the dev repo) being mis-resolved as vault-self, refused as a binding target, or skipped when binding via `--skip-mcp`.
- Write Brain-owned git ignore rules (`.brain/local/` etc.) as scaffold hygiene, independent of MCP scope: `install.sh` calls `init.py --skip-mcp` in the scaffold step so `--skip-mcp`, user-default, and existing-vault installs all get them (fresh installs also inherit them from the template `.gitignore`).
- Add a vault-self registration mode to `_bootstrap/mcp_transport.py` (`apply_mcp_transport_action` + an `init.py --vault-self` flag): write a project-scope `.mcp.json` / `config.toml` with `BRAIN_WORKSPACE_DIR=<vault>` and **no** self-`workspace.yaml`, for both Claude and Codex.
- Add a refuse-guard to `converge_workspace_binding`: binding a vault root raises `WorkspaceBindingError` (`code="vault_root_not_workspace"`), so `setup workspace` / `configure workspace binding` cannot bind a Brain to itself.
- Rework `install.sh` MCP setup into one **exclusive** scope choice — this vault only (project scope, vault-self mode), make this your default brain (user scope + registry default, offering override when a default exists), or skip — never both scopes for one Brain; `--non-interactive` selects this-vault-only. Add an optional `--id <brain-id>` threaded to the machine-registry registration.
- Bump `PROXY_VERSION` to `0.5.2` so the resolver changes take effect on the next proxy restart; move `BRAIN_INSTALL_REF` to `v0.47.0`. `cli/brain` dispatch behaviour and `BRAIN_CLI_VERSION` are unchanged.
- Tests: rung-1 vault-self (including the `AGENTS.md`-only non-vault case), the refuse-guard (including `AGENTS.md`-only not refused), vault-self transport mode for both clients, and the `--id` and non-interactive install paths (`tests/test_resolve_brain_target.py`, `tests/test_vault_self_transport.py`, `tests/test_init.py`, `tests/test_install_script_vault_registry.py`, `tests/test_setup.py`).
