# v0.43.2 — Route brain doctor machine diagnosis through _machine

- Add `src/brain-core/scripts/_machine/` as the launcher-safe machine-management substrate for Phase 1: `discovery.py` discovers current + registered Brains, `topology.py` classifies each Brain's canonical runtime route and shared central runtimes, and `maintenance.py` assembles the machine-level runtime summary that later phases can extend into registry-backed migration and prune workflows.
- Add `src/brain-core/scripts/doctor_machine.py` and teach `cli/brain` to use it when a source Brain is available. `brain doctor` now keeps CLI/PATH/Python survivability checks in Bash, prefers the current Brain as the helper source, otherwise falls back to a registry-discovered Brain, and only uses the old shell-only `~/.brain/venvs/` summary when no helper source is available.
- Keep `brain doctor` honest about failure versus tidiness: orphaned shared runtimes are still reported as cleanup candidates, but they no longer make doctor exit non-zero by themselves. Stale registry entries and unhealthy Brain runtime routes still do.
- Add behaviour-level coverage for `_machine` discovery/topology analysis and for the registry-driven `brain doctor` helper path, alongside runtime/registry regression reruns and the full suite green bar.
- Follow through on the first `_machine` slice: `cli/brain` now falls back to `$XDG_CONFIG_HOME/brain/brains.json` when `vault_registry.py` no longer points at a runnable helper, while schema and version validation remain in the Python-owned `_machine.discovery` loader after handoff.
- Tighten derived machine-registry repair semantics in `src/brain-core/scripts/_machine/discovery.py` and `src/brain-core/scripts/doctor_machine.py`: blocked schemas now fail closed, malformed v1 rewrites take a `brains.json.bak.<timestamp>` backup, stale derived entries remain cheap to prune, and the doctor output now tells the operator to re-run for confirmation after auto-repair.
- Add `src/brain-core/scripts/_common/_paths.py`, cache launcher `python_tag()` resolution in `_common/_venv.py`, and expand `tests/test_machine.py` plus `tests/test_brain_cli.py` to cover malformed registry branches, `ps` failure honesty, `brains.json` fallback, and mixed-version helper selection.
