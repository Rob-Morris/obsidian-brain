# v0.49.4

**Degrade the Obsidian CLI client gracefully on native Windows**

- Guard `obsidian_cli._send` on `socket.AF_UNIX` availability so it reports the Obsidian CLI as unavailable on Windows (where Python lacks `AF_UNIX` and the IPC transport is an unimplemented named pipe) instead of raising `AttributeError` out of the `brain_read environment` read path; callers fall back to the bundled BM25 index.
- Add `tests/test_obsidian_cli.py::TestSend::test_returns_none_when_af_unix_unavailable` pinning the graceful-degradation contract.
- Move `cli/brain` `BRAIN_INSTALL_REF` to `v0.49.4`; `BRAIN_CLI_VERSION` is unchanged because the CLI dispatch surface did not change.
