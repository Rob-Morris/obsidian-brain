# v0.49.0

**Retire init.py and consolidate MCP transport CLI onto configure**

**Transport CLI consolidation**
- Retire `src/brain-core/scripts/init.py` and remove the `brain init` dispatch noun from `DISPATCH_SUBCOMMANDS` in `cli/brain`; `brain init` now falls through to the standard unknown-command path with no redirect or deprecation stub.
- Add `configure mcp --vault-self` (delegating to `_bootstrap/mcp_transport.apply_mcp_transport_action(vault_self=True)`) and `configure workspace bootstrap --remove` (delegating to `cleanup_claude_bootstrap`) so `configure` is the single public transport-management surface. `configure.py` validates CLI arguments and delegates; `_bootstrap/mcp_transport.py` remains the transport/config-write owner.
- Restore the argument-combination guards on `configure mcp`: reject `--user` with `--local`/`--workspace`/`--project`, and reject `--vault-self` with `--user`/`--local`/`--remove` while requiring an explicit `--workspace`/`--project` target.

**Installer cleanup repointed**
- Repoint `install.sh --uninstall` from `init.py` to `configure.py` for recorded project and Claude-local MCP removal, Claude bootstrap-line cleanup (`configure workspace bootstrap --surface claude --remove`), and the user-scope removal instruction; the unavailable-interpreter fallback warning now names `configure.py`.
- `_bootstrap/mcp_transport._remove_bootstrap_line` now reports whether a bootstrap line was removed and raises `InitTransportError` on read/write errors instead of silently swallowing them; `_remove_record` keeps removal best-effort, so a CLAUDE.md cleanup failure no longer aborts MCP teardown or init-state ledger persistence.
- Remove the orphaned `mcp_transport.find_python` and `mcp_transport.ensure_workspace_manifest` helpers, whose only caller was the retired `init.py`.
