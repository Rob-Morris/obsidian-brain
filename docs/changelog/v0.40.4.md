# v0.40.4 - 2026-05-15

**Make build_index.py and search_index.py CLI-only wrappers**

- **Retire the wrappers as Python import surfaces.** `src/brain-core/scripts/build_index.py` and `src/brain-core/scripts/search_index.py` no longer re-export `_search` symbols; they now contain only their CLI entry logic and direct callers toward `_search.index`, `_search.assets`, and `_search.query`.
- **Move wrapper coverage onto real CLI behaviour and canonical owners.** `tests/test_build_index.py`, `tests/test_search_index.py`, `tests/test_process.py`, `tests/test_evaluate_search.py`, and `tests/test_list_resources.py` now import `_search.*` directly, while wrapper coverage runs through `tests/conftest.py::wrapper_cli` subprocess invocations instead of in-process `main()` calls.
- **Tighten the search wrapper contract and guard it structurally.** `src/brain-core/scripts/search_index.py` now uses `argparse` for CLI parsing, and `tests/test_search_modules.py` walks both `src/brain-core/` and `tests/` to assert that no Python module imports `build_index` or `search_index` directly.
- **Align the shipped docs with the converged boundary.** `docs/architecture/overview.md`, `docs/functional/scripts.md`, and `src/brain-core/scripts/README.md` now describe `build_index.py` and `search_index.py` as public CLI entry surfaces only, without transitional hedging.
