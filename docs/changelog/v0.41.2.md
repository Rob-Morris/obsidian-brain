# v0.41.2 — Replace PyYAML with the shared Brain-owned YAML seam

- Replace the shipped `PyYAML` dependency with a Brain-owned standalone YAML subset under `src/brain-core/scripts/_common/_yaml/`, covering the real config/workspace/manifest shapes Brain uses instead of a general YAML runtime dependency.
- Move `config.py`, semantic local-config writes, workspace-manifest reads in `session.py`, and `manifest.yaml` parsing in `sync_definitions.py` onto that shared seam so the runtime no longer depends on importing `yaml`.
- Extract `hash_key()` into a dependency-free shared helper so `generate_key.py` no longer drags config-loading concerns along with it.
- Tighten the managed-runtime bootstrap contract accordingly: managed-runtime required modules now only require `mcp`, and the related convergence/repair/wrapper tests pin the updated contract.
- Add direct support-matrix coverage in `tests/test_common_yaml.py` and migrate config/install/migration tests off `PyYAML` helpers so the supported Brain-owned YAML subset is exercised explicitly.
