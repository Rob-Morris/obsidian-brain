# v0.40.8 - 2026-05-18

**Add duplicate-frontmatter detection, repair, and migration**

- `src/brain-core/scripts/create.py` and `src/brain-core/scripts/edit.py` now reject body-only payloads that start with a frontmatter block, and that guard now also catches the same shape when the block is preceded only by blank lines.
- `src/brain-core/scripts/_common/_frontmatter.py`, `src/brain-core/scripts/check.py`, and `src/brain-core/scripts/_lifecycle/frontmatter_repairs.py` now detect artefacts whose body starts with a second frontmatter block, treat the outer block as authoritative for compliance purposes, salvage inner tags additively, and expose a deterministic normalisation path.
- `src/brain-core/scripts/repair.py` and `src/brain-core/scripts/_repair_runtime.py` now expose `repair.py frontmatter`, a stdlib-only repair scope that scans artefact markdown, preserves the outer canonical frontmatter, unions nested tags, stamps `modified` at repair time, and strips the duplicate block from the body.
- New migration `src/brain-core/scripts/migrations/migrate_to_0_40_8.py` runs during upgrade to repair existing duplicate-frontmatter artefacts automatically, and `brain_read(resource="compliance")` / `check.py` findings now point to `repair.py frontmatter` when they encounter the same corruption.
- `docs/functional/scripts.md`, `docs/user/workflows.md`, and `src/brain-core/scripts/README.md` now document the new repair surface and the broadened compliance guidance honestly.
