# v0.44.4

**Fix brain configure and setup dispatch by accepting top-level --vault**

- Add `make_vault_parent_parser()` to `src/brain-core/scripts/_bootstrap/vaults.py`: a shared parent parser declaring `--vault` with `default=argparse.SUPPRESS`, so an absent `--vault` never clobbers a value already parsed at the top level.
- Wire the shared parent into `src/brain-core/scripts/configure.py` and `src/brain-core/scripts/setup.py` at both the top-level parser and every leaf subparser (`semantic`, `mcp`, `binding`, `metadata`, `bootstrap`, `workspace`), removing the per-subparser `--vault` declarations so the `brain` wrapper's front-injected `--vault <path>` parses before the subcommand name.
- Convert the direct `args.vault` reads in `configure.py` (`semantic` and `mcp` branches) and `setup.py` (`main`) to `getattr(args, "vault", None)` so a suppressed `--vault` falls through to `find_vault_root` auto-detect instead of raising `AttributeError`.
- Amend `docs/architecture/decisions/dd-049-brain-cli-thin-dispatch.md` with the dispatched-scripts contract: every dispatched script must accept `--vault` at the top-level parser, because the wrapper re-injects it before the subcommand.
- Add `tests/test_dispatch_vault_arg.py`: a guard asserting every `brain` subcommand parses the front-injected `--vault` without an argparse error, plus focused tests that `configure` and `setup` accept `--vault` before and after the subcommand without clobber.
- Move `BRAIN_INSTALL_REF` to `v0.44.4`; `cli/brain` dispatch behaviour and `BRAIN_CLI_VERSION` are unchanged.
