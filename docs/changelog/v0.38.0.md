# v0.38.0 — 2026-05-11

**Ship brain CLI and harden launcher/install fallback**

This slice introduces the optional machine-level `brain` CLI and folds in the launcher, installer, and central-runtime hardening that made it reliable in `--skip-mcp`, PATH-based, no-3.12 registry, and Python-minor-churn states. Scripts remain authoritative; the CLI is ergonomic chrome over `.brain-core/scripts/`.

- **Thin brain CLI dispatch layer** — Adopted [DD-049](../architecture/decisions/dd-049-brain-cli-thin-dispatch.md). `cli/brain` resolves the active vault from `--vault`, `$BRAIN_VAULT_ROOT`, or a CWD walk for `.brain-core/VERSION`, then `exec`s the matching `.brain-core/scripts/<name>.py`. `--vault` is normalised to an absolute path before dispatch; all other arguments pass through unchanged. CLI-only commands are `install`, `doctor`, `version` / `--version`, and `--help`; everything else is a thin route into the existing script surface.
- **Launcher and installer hardening** — `cli/brain`, `_common/_venv.py`, and `install.sh` now handle the failure shapes surfaced in repeated review: PATH-discovered launchers are resolved to absolute binaries and version-probed against the 3.12+ floor; `brain doctor` reports both `~/.local/bin` and `/usr/local/bin` installs honestly; `find_runnable_python` falls back through central runtime → legacy `<vault>/.venv` → launcher so `--skip-mcp` vaults still dispatch; `.brain-deps-installed` prevents half-built central runtimes from getting stuck as false positives; and `registry_update` selects an interpreter that can actually run `vault_registry.py`, not just a binary that starts.
- **Compatible-minor central-runtime lookup** — `_common/_venv.py` now reuses an exact-tag runtime first, then any other `>= 3.12` central runtime with the same requirements hash. That keeps `brain` usable across Python-minor churn, especially Homebrew-style `python3.12` → `python3.13` upgrades, without dropping to the bare launcher or reprovisioning a duplicate runtime just because the preferred launcher tag changed.
- **Coverage and metadata** — `tests/test_brain_cli.py`, `tests/test_venv.py`, and `tests/test_install_script_vault_registry.py` cover PATH-discovery fallback, incompatible launchers, half-built runtime recovery, registry fallback probing, and compatible-minor lookup. `pyproject.toml`, `docs/functional/scripts.md`, `docs/functional/cli.md`, `docs/user/user-reference.md`, `docs/user/getting-started.md`, `README.md`, and the DD set now state the 3.12+ launcher/runtime floor consistently.

The CLI is fully optional — `python3 .brain-core/scripts/<name>.py` continues to work exactly as before. Cross-entrypoint runtime convergence stays deferred to `v0.39.0`.
