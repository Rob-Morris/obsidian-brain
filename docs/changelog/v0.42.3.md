- Make `src/brain-core/scripts/_bootstrap/runtime.py:handoff_current_script_to_managed_runtime()` strict again: it now either re-execs or raises, while `preview_managed_runtime()` owns the dry-run preview path used by `src/brain-core/scripts/repair.py`.
- Route `src/brain-core/scripts/repair.py` dry-run through the explicit preview contract, preserve bootstrap steps across managed-runtime re-exec, and stop treating failed bootstrap previews as scope-level planned work.
- Surface `probe_error` through `src/brain-core/scripts/_bootstrap/diagnostics.py:_runtime_state_from_probe()` so `check.py` and repair/runtime diagnostics report why the managed-runtime probe failed instead of collapsing to an empty "unusable" state.
- Remove dead bootstrap-step branch logic from `src/brain-core/scripts/configure.py` and `src/brain-core/scripts/repair.py`; both now trust the handoff summary they were already given instead of re-reading `BRAIN_MANAGED_RUNTIME`.
- Harden git-backed ignore installation in `src/brain-core/scripts/init.py:_git_repo_root()`, `_git_dir()`, and `ensure_brain_ignore_rules()` by using the repository’s `.git` marker instead of locale-dependent stderr matching and by failing explicitly when the chosen ignore destination is unreadable.
- Document the YAML round-trip rule in `src/brain-core/scripts/_common/_yaml/engine.py` that forces integer-shaped strings to stay quoted strings.
- Extend runtime-contract coverage in `tests/test_managed_wrapper_handoff.py`, `tests/test_repair.py`, and `tests/test_configure.py` for the strict handoff path, dry-run preview path, and surfaced probe-error diagnostics.
