# v0.48.1

**Serve unresolved-Brain errors over MCP instead of a bare -32000**

- When the proxy cannot resolve a target Brain at startup — a stale/dangling workspace binding or machine default, or a missing-anchor hard-error — it no longer exits with the actionable cause buried on stderr (which the client surfaces as a generic `-32000 Failed to reconnect`). Instead it runs a degraded MCP server (`_run_degraded_server`): it completes the `initialize` handshake, advertises a single `brain_unavailable` tool whose description carries the cause, and returns the specific, actionable error on every `tools/call` (e.g. "the machine default Brain … is missing or moved — clear the default"). The operator can see and fix the problem, then restart MCP via /mcp.
- Bump `PROXY_VERSION` to `0.5.4` so the runtime drift note prompts a restart onto the degraded-aware proxy.
