# v0.40.1 - 2026-05-14

**Add repair.py runtime and narrow mcp to MCP state repair**

- **Introduce an honest public runtime repair scope.** `src/brain-core/scripts/_repair_common.py`, `src/brain-core/scripts/_lifecycle_common.py`, `src/brain-core/scripts/_repair_runtime.py`, and `src/brain-core/scripts/repair.py` now expose `repair.py runtime` as the public command for repairing the central managed runtime, while `repair.py mcp` is narrowed to current-vault MCP registration repair against a working runtime.
- **Keep runtime repair ownership explicit and lightweight scopes stable.** `repair.py` still owns bootstrap-time managed-runtime repair via `bootstrap_managed_runtime(...)`, `src/brain-core/scripts/_repair_runtime.py` now uses `verify_runtime_post_bootstrap(...)` as the managed-runtime coherence check, and `router`, `lexical`, and `registry` still skip extra package sync beyond the shared interpreter bootstrap guarded by `BOOTSTRAP_SCOPE_MODULES`.
- **Surface the new contract in docs and regressions.** `README.md`, `docs/functional/scripts.md`, `docs/functional/cli.md`, `docs/user/getting-started.md`, `docs/user/user-reference.md`, `docs/architecture/overview.md`, and `src/brain-core/scripts/README.md` now point users at `repair.py runtime` for broken managed-runtime cases, while `tests/test_repair.py` covers runtime bootstrap planning, runtime verification branches, the retained `repair_mcp(...)` Humble Object guard, and the bare-launcher `lexical` dependency boundary.
