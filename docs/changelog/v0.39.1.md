# v0.39.1 — 2026-05-13

**Route MCP warmup progress and guidance through _server_readiness**

This patch finishes the Phase 2 readiness-convergence work started after `v0.35.0`. The coarse MCP readiness contract now has one owner for progress policy and immediate-action guidance, so handler families no longer drift on `router`, `index`, or semantic waits.

- **One owner for coarse MCP readiness policy** — `src/brain-core/brain_mcp/_server_readiness.py` now owns the shared gating and guidance contract for `router`, `index`, and semantic readiness, while `src/brain-core/brain_mcp/_server_runtime.py` exposes `ReadinessInfo` so the policy reads coarse runtime state without reaching back into `server.py` globals.
- **Keep `server.py` as composition root, not policy owner** — `src/brain-core/brain_mcp/server.py`, `_server_session.py`, `_server_reading.py`, `_server_content.py`, `_server_artefacts.py`, and `_server_actions.py` now route warmup-dependent handler flows through the shared readiness owner instead of hand-coding progress decisions or keeping semantic gating as a separate policy seam.
- **Make progress guidance honest across semantic waits too** — `src/brain-core/brain_mcp/server.py` and `src/brain-core/brain_mcp/_server_readiness.py` now derive `next_action` from the blocked capability as well as the coarse warmup state, so semantic progress payloads no longer fall back to the ready-state `Call \`brain_session\`` hint while `brain_search` or `brain_process` is still waiting.
- **Lock the shared contract in with behaviour tests** — `tests/test_mcp_server.py` now asserts shared router, index, and semantic progress payloads across multiple handler families, and also checks that `brain_init` keeps `bootstrap_hint` aligned with the shared guidance owner.
- **Sync shipped MCP/readiness docs** — `docs/functional/mcp-tools.md`, `docs/architecture/overview.md`, and `docs/user/user-reference.md` now describe the shared readiness owner more accurately and include `bootstrap_hint` in the documented `brain_init` response shape.
- **Keep the installer ref aligned with the shipped patch** — `cli/brain` and `tests/test_brain_cli.py` now point `brain install` at `v0.39.1`, so the machine-level installer wrapper stays aligned with `src/brain-core/VERSION` and the new `docs/CHANGELOG.md` row.
