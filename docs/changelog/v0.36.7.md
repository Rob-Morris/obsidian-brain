# v0.36.7

Summary: Restore semantic warmup as a parallel background track

- Start semantic warmup alongside the existing router, index, registry, and session warmup phases so startup can load current embedding sidecars in the background instead of waiting for the first semantic request.
- Teach `brain_mcp/server.py` to apply a full embeddings snapshot from disk when it matches the current router fingerprint, and to mark semantic state dirty for the existing lazy rebuild path when sidecars are missing, stale, or unreadable.
- Gate semantic `brain_search` and `brain_process` requests behind explicit `semantic` progress responses while warmup is still running, while keeping lexical search and context-assembly classification available during the background warmup track.
