# v0.48.2

**Harden MCP startup, managed runtime selection, and client registration repair**

- `brain_mcp/proxy.py` now keeps startup failures inside a degraded MCP session for both unresolved-target and resolved-vault filesystem failures, including proxy-log open errors that previously escaped before the child server existed.
- `brain_mcp/proxy.py` now uses shared `_safe_write_line` and `_parse_jsonrpc_line` helpers so normal and degraded proxy loops agree on client disconnects, malformed frames, and non-object JSON-RPC frames.
- `_bootstrap/workspace_binding.py` now classifies manifest and registry read failures with `WORKSPACE_ERROR_FILESYSTEM_ACCESS`, letting `proxy.py` choose filesystem remediation instead of binding/default remediation.
- `brain_mcp/proxy.py`, `docs/functional/mcp-tools.md`, `docs/architecture/overview.md`, and `dd-028-version-drift-detection.md` now use generic "restart MCP" guidance instead of Claude-specific `/mcp` restart wording.
- `_bootstrap/runtime.py` now compares executable paths without collapsing symlinks so managed virtualenv Python paths do not get mistaken for the system interpreter during runtime handoff, and `_common/_venv.py` now resolves bare launcher names through `PATH` before deriving central runtime paths.
- `brain_mcp/proxy.py` now avoids `select.select()` on Windows child stdout pipes, so Windows MCP initialize responses are read instead of wedging behind an unread pipe. Unix keeps timeout-based hung-child detection; Windows keeps EOF/crash recovery but not pipe timeout hang detection.
- Claude SessionStart hooks now use the same managed Python as the Brain MCP config, use Windows command quoting on Windows, and repair replaces stale machine-Python hooks instead of appending duplicates.
- `check.py`/`repair.py` now surface and repair common Brain MCP registration drift, including Claude/Codex configs pointing at system Python and stale or duplicate Claude SessionStart hooks. `migrate_to_0_48_2.py` converges affected project registrations during upgrade.
- `README.md`, `AGENTS.md`, `docs/user/getting-started.md`, and `docs/functional/scripts.md` now clarify the front-door documentation, agent routing, MCP scope examples, and the `retrieval_embeddings.py` script surface.
- Tests cover degraded-mode filesystem guidance, custom lead/guidance text on the MCP wire, non-object JSON-RPC frames, broken-pipe handling, and filesystem-coded workspace manifest failures (`tests/test_proxy_degraded_mode.py`, `tests/test_mcp_proxy.py`, `tests/test_workspace_binding_heal.py`).
- Bump `PROXY_VERSION` to `0.5.6` and move `BRAIN_INSTALL_REF` to `v0.48.2`; `BRAIN_CLI_VERSION` is unchanged because the CLI dispatch surface did not change.
