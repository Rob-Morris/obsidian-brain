# Standards

Reusable patterns and conventions that are generic enough to apply beyond this project. Each standard is a self-contained document describing a pattern, its rationale, and how to adopt it.

## When to Create a Standard

When a pattern emerges from project-specific work but has no inherent dependency on the project. If the pattern could be copy-pasted into an unrelated repo and still make sense, it belongs here rather than in `docs/`.
